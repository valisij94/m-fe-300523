## Практическое занятие №1

### Тема: React.js. Работа с библиотекой React Router.

Репозитарий содержит материалы по практическому занятию №1, посвященному работе с библиотекой React Router.

### Цели занятия
- закрепление знаний, полученных на лекции;
- разбор практических примеров;
- практическое применение полученных знаний.

### Предметная область
Работаем в контексте маленькой социальной сети - создаем посты, оставляем комментарии, и так далее. Приложение будет содержать несколько страниц (в этом нам поможет React Router).

При входе пользователь попадает на главную страницу. Здесь будет заголовок с названием нашей соцсети, текст-описание (чем мы хороши), и две кнопки: "Вход" и "Регистрация".
При нажатии на кнопку "Вход", мы должны показать ему форму входа.
При нажатии "Регистрация" - открыть форму регистрации.

Еще у нас есть страница "Лента", со списком постов. И страница создания нового поста.

### Задачи занятия
1. Пристрелочная задача. Содзаем папку `components`, в ней создаем папку `button`. Там нам нужно создать компонент `Button` для рендеринга кнопки. В пропсах мы будем передавать текст кнопки `buttonText`, обработчик события клика `clickHandler`, и флаг `disabled` - разрешена кнопка или нет.
2. Создаем CSS-модуль для нашей кнопки. Класс для кнопки - `filledButton`. Нужно, чтобы у кнопки были:
 - цвет фона - #0096CD
 - цвет текста - белый
 - внутренние отступы - 10 пикселей
 - закругленные углы (10 пикселей радиуса)
 - убрать внешнюю рамку кнопки
 - при наведении на кнопку, цвет фона должен меняться на `blue`, время перехода - 0.5 секунды.
3. Делаем структуру страниц приложения. Создаем в папке `src` папку `pages`. В ней будем создавать компоненты-страницы нашего приложения. Создадим там папки для страниц: главная (`Home`), страница регистрации (`RegisterPage`), страница постов (`FeedPage`), и страница создания нового поста (`NewPost`). Делаем главную страницу. На ней должны быть заголовок с названием (`The Best Social Network`), текстовка, и две кнопки. Для кнопок используем наши компоненты (`Button.jsx`).
4. Рисуем форму входа. Она должна содержать инпуты для логина (`username`), пароля (`password`), и кнопку "Войти". Значения инпутов нужно хранить во внутреннем стейте (`state`) формы.
5. Рисуем страницу с формой регистрации. Она должна содержать инпуты для логина (`username`), пароля (`password`), емейла (`email`), имени пользователя (`name`) и также содержать кнопку "Зарегистрироваться". Данные инпутов нужно хранить во внутреннем состоянии формы (`state`).
6. Добавляем обработчик. При нажатии на кнопку регистрации - нам нужно отправить POST-запрос на бэк, по адресу `https://jsonplaceholder.typicode.com/users`. В теле запроса нам нужно передать `username`, `name`, `email` - с такими же ключами. В ответ нам придет JSON-объект с идентификатором пользователя - этот объект нужно вывести в консоль.
7. Теперь добавим маршрутизацию в наше приложение. Нужно, чтобы по маршруту `/login` была бы страница входа, а по маршруту `/register` - страница регистрации. Нам нужно:
 - обернуть приложение в `BrowserRouter`
 - создать маршруты (лучше - в компоненте `App`)
8. Немного освежим в памяти хуки роутера. На странице регистрации, нам нужно, чтобы при успешной регистрации (если нам пришел ответ с ID пользователя), перебрасывать пользователя на страницу входа.
9. Готовимся рисовать список постов. В папке `components` в соответствующих папках, создаем компоненты `PostsList`, `PostCard` для списка постов, карточки поста соответственно.
10. Рисуем страницу с лентой. Она будет содержать ленту постов. При заходе на страницу, она должна обратиться "к бэку" за списком постов. Это адрес `https://jsonplaceholder.typicode.com/posts`. При получении результата - рисуем список постов. Посты рисуем в виде компонентов, каждый пост оборачиваем в ссылку `Link` на страницу подробной информации о посте (ее создадим позже). Пока что в качестве адреса ссылки указываем корневую страницу (`/`). Страницу нам нужно также добавить в маршруты приложения, по адресу `postsPage`.
11. Теперь создаем страницу просмотра подробной информации о посте (`PostPage`). Тут будет заголовок, текст поста, и список комментариев к нему. Пока это все. И эту страницу тоже добавляем в маршруты приложения. Нам нужно, чтобы мы передавали в УРЛе номер поста, и наш адрес был бы вида `/postsPage/1`.
12. Дорабатываем ссылки на странице ленты - при клике по посту, надо перебрасывать пользователя на страницу просмотра подробной информации о посте.
13. При заходе на страницу подробной информации о посте, нам нужно запросить информацию о посте по адресу `https://jsonplaceholder.typicode.com/posts/1`. ID поста нам предстоит взять из УРЛ (передадим в параметрах). Тут нам тоже поможет один из хуков библиотеки React Router.
